% POSIZIONE:
% È la porzione di codice situata DOPO subito dopo gli eventuali Magic Comments
% dell'editor, ma PRIMA della dichiarazione della classe (\documentclass).
%
% SCOPO E DEFINIZIONE:
% Il Pre-Preambolo è un ambiente "sterile". In questa fase, il motore LaTeX
% è appena stato avviato: non conosce ancora le dimensioni della pagina,
% i font o la struttura del documento. Lo scopo di questa sezione è agire
% sulla gestione delle dipendenze in memoria, depositando istruzioni che
% il compilatore dovrà ricordare ed eseguire in un secondo momento.
% Si utilizza principalmente per patching, o per controllo di alcune dipendenze,
% per workarounds, o per modificare globalmente il comporamente di alcuni
% elementi.
%
% CASI D'USO COMUNI:
% Poiché in questa zona è vietato inserire testo visibile, formattazioni
% o comandi come \usepackage, si utilizzano solo primitive a basso livello,
% in precisione è possibile usare solo un ristretto insieme di comandi.
% Comunque i casi d'uso più comuni sono i seguenti:
%
% 1. Iniezione di opzioni (\PassOptionsToPackage{...}{...}):
%    Permette di forzare o aggiungere opzioni a pacchetti che verranno
%    caricati più avanti e "alla cieca" dalla classe di documento (es.
%    per evitare errori di "Option clash").
% 2. Architettura del PDF (\DocumentMetadata{...}):
%    Per configurare l'infrastruttura profonda del file (es. creazione
%    di PDF/A per l'archiviazione a lungo termine o PDF accessibili).
% 3. Controlli di sicurezza e debug (\NeedsTeXFormat, \listfiles):
%    Per imporre una versione minima del compilatore o tracciare nel file
%    di log tutti i percorsi esatti dei pacchetti letti dal sistema.
%
% ==============================================================================

\PassOptionsToPackage{normalem}{ulem}   % Fix per Bug Overfull hbox in
                                        %bibliografia causato dal pacchetto ulem


% ==============================================================================
% BUGFIX: OVERFULL HBOX IN BIBLIOGRAFIA E CONFLITTO PACCHETTO ULEM
% ==============================================================================
% COMANDO: \PassOptionsToPackage{normalem}{ulem}
% POSIZIONE CORRETTA: Pre-Preambolo (Rigorosamente PRIMA di \documentclass)
%
% 1. IL PROBLEMA (OVERFULL \HBOX IN BIBLIOGRAFIA)
% Durante la compilazione, i titoli lunghi dei libri nella bibliografia
% causavano gravi warning di "Overfull \hbox", sbordando oltre il margine
% destro della pagina. Inoltre, i titoli apparivano sottolineati anziché
% nel classico corsivo tipografico standard.
%
% 2. LA CAUSA (SOVRASCROTTIRA DI \EMPH)
% La custom class del progetto (HKNdocument.cls) carica internamente il
% pacchetto 'ulem' tramite il comando \RequirePackage{ulem}. Di default,
% questo pacchetto sovrascrive i comandi standard \emph e \em, sostituendo
% il corsivo con la sottolineatura continua, precisamente sostituendoli con
% lo stesso comando \uline propsto dal pacchetto ulem stesso. Questo vincolo
% rigido impedisce al motore di sillabazione di LaTeX di spezzare la
% frase a fine riga quado viene usato \emph, in quanto è generato come un box
% insecabile, generando l'errore di overfull.
%
% 3. LA SOLUZIONE E L'EFFETTO DI 'NORMALEM'
% L'opzione 'normalem' (Normal Emphasis) istruisce il pacchetto 'ulem' a
% non sovrascrivere il comportamento di \emph, ripristinando i corsivi
% originali e, di conseguenza, la corretta sillabazione a fine riga.
% Questo non causa alcuna perdita di features, visto che \uline è sempre
% provvisto dal pacchetto.
%
% 4. NESSUN EFFETTO COLLATERALE NELLA RISOLUZIONE DEL BUG
% Analizzando la classe HKNdocument.cls, si evince che il pacchetto ulem
% è stato caricato dal project manager al solo scopo di definire la macro
% \myuline. I corsivi della classe sono stati realizzati in modo sicuro tramite
% \textit. Questa iniezione è quindi priva di effetti collaterali: corregge il
% bug della bibliografia senza alterare in alcun modo l'estetica o le macro
% progettate per l'associazione.
%
% 5. PERCHE' NEL PRE-PREAMBOLO? (LA GESTIONE DELLA MEMORIA)
% Questo comando deve stare prima del \documentclass. Se posizionato nel
% preambolo standard, LaTeX lo ignorerebbe o andrebbe in crash ("Option
% clash"), perché la classe avrebbe già caricato il paccheto ulem in memoria.
% Posizionandolo nel Pre-Preambolo, si lascia un "post-it" al compilatore:
% quando la classe chiamerà ulem, LaTeX gli inietterà di nascosto l'opzione
% 'normalem', aggirando i blocchi di sistema.
%
% 6. ALTRA SOLUZIONE NON PRATICA: MODIFICARE LA CUSTOM CLASS
% Ovviamente per risovlere basterebbe modificare la custom class HKNdocument.cls
% aggiunenddo l'opzione 'normalem' al comando \RequirePackage{ulem}. Tuttavia
% questa soluzione invaliderebbe l'uncità dei file della classe pensata per il
% progetto. Tuttavia si tratta di un bug meritevole di correzione in generale,
% attraverso una pull-request.
%
% 7. ALTRE SOLUZIONI INTERESSANTI:
% - Si avrebbe potuto risolvere anche sfruttando il comando \normalem del
%   pacchetto ulem, che ripstina il comando \emph al suo comportamento normale,
%   in due modi diversi:
%   1.  Mettendolo prima della bibliografia:
%       \normalem  % Comando: Ripristina \emph al suo stato originale (corsivo)
%       \printbibliography
%       \ULforem   % Comando: Riattiva la sottolineatura per \emph (se serve)ù
%   2.  Sfruttando un hook, ossia un'iniezione di codice:
%       \AtBeginBibliography{\normalem}     % lancia \normalem prima di stampare
%                                           % la bibliografia
% - In modo simile, si sarebbe potuto sfruttare un backup del comando \emph
%   originale, fatto con \let\originalemph\emph nel pre-preambolo, per poi
%   risovrascriverlo prima della bibliografia, con \let\emph\OriginalEmph.
% ==============================================================================

\PassOptionsToPackage{english}{babel}   % Fix per Bilinguismo, non supportato
                                        % nativamente da HKNDocument
                                        % Carica inglese come lingua secondaria
                                        % Necessario per Bibliografia bilingue

% ==============================================================================
% INIEZIONE DELLA LINGUA SECONDARIA (INGLESE) SENZA MODIFICA DELLA CUSTOM CLASS
% ==============================================================================
% L'istruzione \PassOptionsToPackage modifica la chiamata del pacchetto Babel,
% che avviene nella custom class HKNdocument, chiamata da \documentclass.
%
% SCOPO: BIBLIOGRAFIA CONTENENTE FONTI BILINGUE
% Questa pre-configurazione è un prerequisito tecnico necessario per il
% pacchetto 'biblatex' (se inizializzato con il parametro 'autolang=other').
% Assicura la disponibilità in memoria dei dizionari di sillabazione
% (hyphenation) e delle stringhe di localizzazione anglosassoni, vincolanti per
% il corretto parsing e rendering delle voci bibliografiche recanti il metadato
% 'langid={english}'.
%
% IDEA DI FONDO:
% Il comando inietta la lingua 'english' nella lista delle opzioni destinate
% al pacchetto 'babel', disaccoppiando il caricamento dalle restrizioni della
% classe custom, che prevede una sola lingua.
%
% WORKAROUND SFRUTTATO:
% 1. La classe 'HKNdocument' implementa logiche condizionali (es. \ifx)
%    basate sulla variabile \DocumentLanguage. Passare una lista concatenata di
%    lingue direttamente alla classe corromperebbe l'esecuzione di tali script
%    (inclusa la generazione del blocco licenze).
% 2. Demandando l'opzione tramite \PassOptionsToPackage, il compilatore LaTeX
%    memorizza l'argomento. Al momento dell'effettiva invocazione di 'babel'
%    da parte della classe (es. \RequirePackage[italian]{babel}), avviene un
%    merge delle opzioni.
% 3. L'elenco finale risolto dal compilatore risulta essere [english, italian].
%    In accordo alle direttive di 'babel', l'ultimo parametro definisce
%    la Main Language del documento, lasciando le lingue precedenti disponibili
%    in memoria come Secondary Languages.
%
% ============================================================================
